The Model

Graph Neural Network
Architecture

At its core, MolPredict uses a Graph Neural Network (GNN) that treats each molecule as a mathematical graph: atoms are nodes and chemical bonds are edges. This representation is far richer than traditional fingerprints because the model can learn structural and relational patterns simultaneously.

The model was trained using PyTorch Geometric and saved as a complete model object — meaning the architecture and weights are bundled together for easy deployment. At inference time, it runs in eval() mode with torch.no_grad() for maximum speed.

Feature Engineering

What the Model Sees

Each atom is encoded as a 6-dimensional vector. These hand-crafted features capture the chemical identity and environment of every atom in the molecule.

Dimension Feature Description Type
0 Atomic Number Identifies the element (H=1, C=6, N=7 …) Integer
1 Degree Number of bonds to neighboring atoms Integer
2 Formal Charge Net charge on the atom (can be negative) Integer
3 Hybridization sp, sp2, sp3 orbital state (cast to int) Enum→Int
4 Aromaticity 1 if part of an aromatic ring, else 0 Binary
5 Total H Count Explicit + implicit hydrogen atoms attached Integer

Inference Pipeline

From Request to Response

1. HTTP Request

Client POSTs JSON {"smiles": "..."} to /api/predict. Flask validates the payload.

2. SMILES Validation

RDKit's Chem.MolFromSmiles() parses the string. Invalid SMILES returns an immediate error response.

3. Graph Construction

Node feature matrix X (N×6) and edge index (2×E) are extracted. A batch vector of zeros handles the single-molecule batch.

4. Device Transfer

Tensors move to GPU if CUDA is available, otherwise CPU. The model was loaded at startup and stays resident in memory.

5. Forward Pass

GNN runs model(x, edge_index, batch_index) under torch.no_grad(). Argmax gives the class; softmax gives probabilities.

6. JSON Response

Result: {"smiles", "prediction", "confidence"} is returned to the client in under 15 ms.
